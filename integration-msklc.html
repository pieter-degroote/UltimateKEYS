UltimateKEYS Keyboard Layout - MSKLC (Windows)

International US QWERTY layout with enhanced multilingual support for AutoHotkey (Win), MSKLC (Win) and XKB/Xmodmap (Linux).

Integration in Windows via MSKLC

  • The installer can be found in the Core repository.  
  • In case a previous version has been installed, that version needs to be removed beforehand via Control Panel -> Programs and Features. Then a reboot is required and afterwards, another version can be installed instead.
  • Note that this is a slimmed-down version, without Compose Key. Optionally, this can be added by running the AutoHotkey script on top of the MSKLC version of UltimateKEYS (see instructions).
  • Another option is to use the Emoji Picker :  Press Win Key + Period (.) or Win Key + Semicolon (;) to insert the desired emoji, symbols,...

Important - Known limitations in Windows

When changing the Windows/Microsoft password, using another keyboard layout can INTERFERE in some cases where your keystrokes are either registered via the other layout or as standard US QWERTY (depending on the keyboard layout during input).

On the login screen, the keyboard layout might be DIFFERENT, which can result in the WRONG input !

To be "better safe than sorry", you should ALWAYS make your password VISIBLE during modification, to avoid being locked out of your account !

=> Win/Linux :  Only use "BASIC LATIN" characters (codes 32 to 126) for your Windows, Microsoft Live and Linux passwords !
Otherwise, it might be impossible to type any Unicode character higher than "Basic Latin" at the login screen/prompt !  =>  ACCOUNT LOCKOUT

=> This because another keyboard layout might be active at the login screen (mostly US QWERTY), and the needed keyboard (with the additional higher Unicode characters) might NOT be present there in the list of active keyboard layouts !
