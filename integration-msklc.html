UltimateKEYS Keyboard Layout - MSKLC (Windows)

Integration in Windows OS using MSKLC (not recommended on Win 10 and/or in a production environment)

Important - Please note :

  • I recommend using the AutoHotkey scripts as these are a less radical approach to the Windows OS.
  • I do not recommend deploying the MSKLC version in a production environment.
    This because Microsoft Keyboard Layout Creator 1.4 is older software (published on 2007-01-30) which has not yet been revised for upstream compatibility and integration with newer Windows versions (and/or updates and future upgrades).
  • Alternatively, KbdEdit can be used for opening and using MSKLC source files.
  • The MSKLC source files can be found in the 'msklc' folder of this repository.

 

Important - Known limitations in Windows

When changing the Windows/Microsoft password, using the SW variant (and other keyboard layouts) can INTERFERE in some cases where your keystrokes are either registered via the SW variant (and other layouts) or as standard US QWERTY (depending on the keyboard layout during input).

On the login screen, the keyboard layout might be DIFFERENT, which can result in the WRONG input !

To be "better safe than sorry", you should ALWAYS make your password VISIBLE during modification, to avoid being locked out of your account !

=> Also make sure to use only "LOWER ASCII" characters (codes 32 to 126) for the Windows/Microsoft password. Otherwise, it might become impossible to type any other higher Unicode characters at the login screen/prompt !

=> This because another keyboard layout might be active at the login screen (mostly US QWERTY) and the desired keyboard (with the additional higher Unicode characters) might NOT be present there in the list of keyboard layouts !
